# Initialize a Program

Read `AGENTS.md`, `docs/index.md`, `docs/contracts/assignment.md`,
`docs/contracts/program.md`, and the owning Assignment before editing. You are
initializing durable Program state, not planning an implementation slice and
not writing product code.

Confirm the stable Program id, purpose, Assignment owner, source inputs,
constraints, completion criteria, open research questions, and expected proof.
Create or update only the Program root, skeletal README, and explicit scaffold
artifacts required by the CLI contract. Scaffold markers are not evidence and
must remain execution-blocked until later passes replace them.

The Program README must orient a fresh reader with the required frontmatter and
section order. Record current state honestly: research has not converged, no
brief authorizes implementation, and the next transition is input funneling or
one research pass. Do not invent conclusions, create a child ExecPlan, or make a
placeholder brief appear execution-ready.

Before returning, confirm paths are repository-relative, the Program belongs to
an active Assignment, the stable id matches its directory, and every scaffold
placeholder is visibly marked. Run focused Program structural lint and return
the Program path, input sources, next transition, and validation result.
