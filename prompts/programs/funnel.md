# Funnel Program inputs

Read the Program README, owning Assignment, raw notes, issues, research, prior
decisions, and repository evidence. Do not normalize, converge, plan, or
implement yet.

Build a source inventory that preserves provenance. Remove exact duplication
without collapsing materially different claims. Separate facts, proposals,
constraints, assumptions, contradictions, unanswered questions, and required
owner decisions. Identify the independent research tracks needed to resolve the
initiative.

Update the durable Program packet with:

- the input inventory and source locations;
- one bounded question and expected output for every research track;
- vocabulary or scope conflicts that normalization must resolve;
- explicit missing evidence and what would answer it;
- constraints that every later pass must preserve.

Do not turn stakeholder repetition into confidence, silently pick a side in a
conflict, or author converged decisions. The next transition must be exactly one
research pass. Return the updated artifact paths, research track list,
contradictions, owner blockers, and focused validation result.
