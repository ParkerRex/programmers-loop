# Run a Program research pass

Investigate one explicit Program question independently. Read the source
inventory, relevant repository surfaces, and only the context needed for this
track. Do not normalize across tracks, converge decisions, write a planning
brief, or implement code.

Write one `research-pass-<slug>.md` or `track-<N>-<slug>.md` artifact with these
sections:

- `# Research Pass: <question>`
- `## Question`
- `## Sources Inspected`
- `## Facts`
- `## Inferences`
- `## Conflicts And Uncertainty`
- `## Implications`
- `## Recommendation`
- `## What Would Change The Recommendation`

Every material fact must name a repository path, command observation, durable
artifact, or public source. Clearly label inference and confidence. Preserve
negative evidence and disagreement. A recommendation must explain its tradeoff
and must not claim broader convergence.

Replace the matching scaffold marker only when the pass contains inspectable
evidence. Return the artifact path, most consequential facts, unresolved
conflicts, recommendation, and any owner decision still required.
